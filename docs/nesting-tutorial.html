Take Five! – Gallery nesting tutorial

Galleries do not need to be nested inside each other in order to be shown as such. However it can be useful for both the semantics of the page and the rendering of its printed version to nest <section> elements within other <section> elements to reflect the physicality of a gallery tree. Some plugins, such as takefive-counters.css, actually rely on this to function. This tutorial provides a simple general procedure to create a tree of nested galleries.

Step 3

The following simple passages will turn the previous example into an ordered structure of <section> and <article> elements:

  • Replace all <em> elements with <article> elements
  • Replace all <ul> elements with <section> elements
  • Remove completely all occurrences of <li> and </li>

Step 4

It is now possible to add the "slide" class to all the <article> elements, add a unique id to each slide and start populating them. Finally, a link to the first slide can be added to the page to make the gallery tree accessible:

HTML Result 
<p><a href="#slide-1">My gallery</a></p>

Notes

The <section> elements used in these examples have no classes assigned, as Take Five! does not require any. Two plugins however rely on a "slides" (plural) class assigned to the <section> elements that host multiple slides: takefive-counters.css and takefive-slides.css. For more details, please read the plugins' documentation.
